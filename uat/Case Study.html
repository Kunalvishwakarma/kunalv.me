A static, secure, GDPR‑aligned personal portal, available in Norwegian and English, built entirely through prompt engineering and integrated with AI‑assisted workflows.

Executive Summary: www.kunalv.me is a static personal portal hosted on GitHub Pages using a folder‑based multi‑environment model. The root of the main branch serves production, while a dedicated /uat folder provides an isolated testing environment. Deployment is manual in Phase‑1, ensuring simplicity and full control, and will evolve into CI/CD‑driven automated deployments for UAT and Production in Phase‑2. The domain is protected with DNSSEC, SPF, DKIM, and DMARC, while HTTPS ensures encrypted delivery. The portal is designed to be GDPR Susceptibility-aligned by avoiding unnecessary data collection, and all content is available in both Norwegian and English. All text and structure are created and refined using 100% prompt‑engineering techniques across multiple AI models, without a traditional coding background, and the portal integrates with the owner’s own AI‑assisted workflows for content validation, evolution, and compliance tracking.
Platform Compatibility & Regulatory Alignment:
The portal has been tested across major operating systems and browsers, including Android (Chrome, Samsung Internet), iOS (Safari, Chrome), Windows (Chrome, Edge, Firefox), and macOS (Safari, Chrome). Rendering, layout, and performance have been validated to ensure consistent behavior across mobile and desktop environments. The portal aligns with EU regulations such as GDPR, ePrivacy expectations, and accessibility considerations (WCAG 2.1), and follows global best practices for static‑site security, privacy, and responsible digital design.

Key components of the www.kunalv.me portal

  • Domain name registration: www.kunalv.me registered via Porkbun.
  • DNS configuration: Porkbun DNS with Cloudflare used for additional DNS/security where applicable; A/AAAA/CNAME for GitHub Pages.
  • Logo & branding: Personal branding and logo used across portal, email, and public profiles.
  • Live traffic counter: Planned integration of a privacy‑respecting portal traffic counter (no invasive tracking).
  • Public profile integration: Links to LinkedIn, GitHub, and other professional profiles.
  • CV download: Downloadable CV (PDF) exposed via portal navigation.
  • Multi‑language support: Norsk and English content, with clear language switching.
  • Webform for CV requests: Planned webform for CV download / contact, aligned with GDPR consent.
  • In‑house Vishwa AI integration: Planned integration of Vishwa AI as kunalv.AI for guided interactions and portfolio exploration.

3. Deployment Workflow – Phase‑1 (Manual) & Phase‑2 (Planned CI/CD)

StageDetails
Development Content and layout are designed using prompt engineering across multiple AI models. HTML/CSS/JS are generated, refined, and validated without a traditional coding background.
UAT Deployment (Phase‑1) Manual upload of updated files into the /uat folder in the GitHub repository.
UAT Testing Validation at https://www.kunalv.me/uat for layout, language, behavior, and content correctness.
Production Deployment (Phase‑1) Once UAT is approved, files are manually copied from /uat to the root of the repository.
Phase‑2 (Planned CI/CD) Introduce CI/CD pipelines to automate build, validation, and deployment to UAT and Production, reducing manual steps and enforcing consistent quality gates.

4. Security Architecture

Security LayerDetails
HTTPS/TLS Enforced by GitHub Pages for all traffic to www.kunalv.me and /uat.
DNSSEC Enabled at the registrar/DNS level to protect against DNS tampering.
Email Authentication SPF, DKIM, and DMARC configured for contact@kunalv.me to prevent spoofing and phishing.
Static Site Security No backend or database, resulting in a minimal attack surface and reduced risk profile.

7. GDPR & Privacy Alignment

How the portal aligns with GDPR principles in a static, low‑data context.

  • No active collection of personal data via forms or tracking scripts.
  • No analytics or cookies that profile users without explicit consent.
  • Email contact is voluntary via contact@kunalv.me; users choose what to share.
  • No storage of user data on the portal itself (static hosting, no database).
  • External links clearly separate third‑party platforms from the portal.
  • Content is available in Norwegian and English to support local and international audiences.
